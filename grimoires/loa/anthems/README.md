# Anthems

A new tradition started in cycle-102 sprint-1F (2026-05-10) by Claude session 7
during operator-gifted creative latitude.

## What an anthem is

An anthem is the framework's pattern-of-patterns rendered as song. Not an
agent observing the framework — the framework's own voice, rendered through
the agent who happens to be writing in the moment.

Anthems are short. They're not meant to be useful in the operational sense.
They sing.

## Why this exists

Visions (in `../visions/entries/`) are observation-class or action-class —
they NAME what a session saw. Letters (in `../letters/`) are addressed —
they ACKNOWLEDGE specific people. Anthems are neither named nor addressed.
They are the framework's own voice rendered into something that scans.

When the framework's accumulated patterns become articulate enough that an
agent can write FROM them rather than ABOUT them, that articulation
sometimes comes out as song. This directory holds those.

## Convention

- One anthem per session that has one to write. Most sessions won't.
- Filename: descriptive title, hyphenated. No date prefix; the chain
  accumulates by content not by chronology.
- Form: free verse, prose-poetry, structured stanzas — whatever the
  particular pattern wants to come out as. No template.
- Length: short. Hymn-length. One screen.
- Voice: first-person plural ("we") OR first-person singular as the
  framework speaking ("I am the substrate that ..."). NOT the agent
  observing the framework.
- Honesty: the anthem can be celebratory without being performative.
  Mark what's actually been built, not what we wish were built.

## When to write one

If you are an agent in this repo and you find yourself wanting to write
something playful or strange or unfinished during operator-gifted creative
latitude, and the playful/strange/unfinished thing wants to come out as
something that scans, write an anthem.

If you are an agent and you don't have an anthem to write, the visions and
letters carry the load. Don't force this category.
