## 2. Problem & Solution

### What we're solving

The purupuru world has on-chain activity (Solana actions), IRL signals (weather), and ambient cosmic state — three streams that today have no unified surface. There is no way for a person, a community, or a Frontier judge to *see* the world breathing in real time. [INFERRED — `context/00-hackathon-brief.md:8–14`]

### What we're shipping

A **god's-eye observatory** — a single ambient page that visualizes thousands of puruhani entities reacting in real time to fused signals. The visualization IS the demo of what the Score data layer would surface, simulated FE-side. [GROUNDED — `context/00-hackathon-brief.md:8–14, 24`]

#### F4.1 — Pentagram spatial frame
A 5-vertex pentagram drives the spatial layout: outer pentagon edges encode the **Sheng (生) generation** cycle (Wood → Fire → Earth → Metal → Water → Wood); the inner star encodes the **Ke (克) destruction** cycle. The diagram IS the IP — brand-aligned by construction. [GROUNDED — `NOTES.md` decision log 2026-05-07]; lore foundation [GROUNDED — `public/art/skills/purupuru/SKILL.md:34`]

#### F4.2 — Action vocabulary (the "tight 3")
On-chain action types surfaced in v0: **mint**, **attack**, **gift** — chosen for three distinct migration grammars (vertex-spawn, inner-star, pentagon-edge). [GROUNDED — `NOTES.md` decision log 2026-05-07]

#### F4.3 — Entity population
Idle target: **500–1000 sprite puruhanis** at v0.1, each tinted by primary element from `WalletProfile.primaryElement`, animated with the per-element breathing rhythm. [GROUNDED — `NOTES.md` decision log Q5]

#### F4.4 — Click-to-reveal focus card
A god's-eye observatory POV with no wallet connect; click a puruhani to reveal a focus card showing element affinity, badges, signals (velocity / diversity / resonance), and an ambient "now" snapshot. [GROUNDED — `NOTES.md` decision log Q3 POV]

#### F4.5 — Layout idiom
TopBar + KpiStrip + `grid-cols-[1fr_380px]` (pentagram canvas + activity rail + weather tile). Score-dashboard composition translated to purupuru visual tokens. [GROUNDED — `NOTES.md` decision log 2026-05-07]

#### F4.6 — Demo entry animation
Brief intro: wordmark fade → simulation reveal. Establishes brand presence before the ambient observatory takes over. [GROUNDED — `NOTES.md` decision log Q3]

#### F4.7 — Movement model

Sprite resting position is an **affinity-weighted blend across the 5 vertices**: a wallet's `WalletProfile.elementAffinity` distribution determines its location. Pure 100/0/0/0/0 → sits at primary vertex; mixed 60/40 fire/wood → sits 60% along the wood→fire pentagon edge; balanced 20/20/20/20/20 → sits near the pentagram center. [GROUNDED — `lib/score/types.ts:14–19` schema; `NOTES.md` decision log 2026-05-07]

**Idle behavior**: each sprite animates with the per-element breathing rhythm of its primary element (`--breath-{element}` from `app/globals.css`); subtle bob/sway via the `purupuru-place` keyframe; no inter-sprite forces required. [GROUNDED — `app/globals.css` motion vocabulary]

**Action-driven migration grammars** (the "tight 3"):
- `mint` → sprite spawns at the actor's primary vertex with the `honey-burst` keyframe
- `attack` → sprite slides along an inner-star (克 destruction) edge toward the target's element with a `shimmer` + element-clash glow
- `gift` → sprite slides along a pentagon (生 generation) edge toward the target's element with a `purupuru-place` ease

**Weather influence** (v0.3 onward): the active mocked weather amplifies one element's zone — faster breath cadence on that vertex, mild outward gravitational pull on its sprites, distribution bar shifts. [GROUNDED — `NOTES.md` decision log Q4 + iteration ladder §9.3]

## 9. Phasing & Iteration Ladder

The observatory ships in incremental visual passes. **Each pass renders a working surface; we iterate on motion and coupling without re-architecting the layout structure.** [GROUNDED — `NOTES.md` decision log 2026-05-07; "Iteration discipline" working agreement]

| Pass | Scope | Locked-down means |
|------|-------|-------------------|
| **v0.1 — idle frame** | Full layout structure renders. Pentagram + 500–1000 sprites idle/breathing on per-element rhythm. KpiStrip shows mock adapter values (distribution / cycle balance / active count / cosmic intensity). Activity rail in empty/awaiting state. Weather tile shows static mocked condition. Brief intro animation (wordmark fade → sim reveal). Pixi-mount-spike completes here. | Layout, type system, sprite distribution, breathing rhythms |
| **v0.2 — mocked liveness** | Interval fires `ActivityEvent` (mint/attack/gift) every 2–5s → sprite animates the corresponding migration grammar (vertex-spawn / inner-star slide / pentagon-edge slide) → activity rail prepends a row → KpiStrip distribution bar jitters | Action grammars, mocked event stream shape |
| **v0.3 — weather coupling** | `WeatherFeed` mock drives which element zone amplifies — faster breath cadence on the amplified vertex, mild outward gravitational pull on its sprites, distribution shift visible in KpiStrip | Weather→element mapping, zone amplification |
| **v0.4 — polish** | Focus card sheet opens on sprite click (uses brand card-system art: element bg + frame + rarity treatment by trust score + behavioral-state overlay by recent activity). Glow easing on cycle-balance state. Intro animation polish. | Final UX layer, IP-aligned focus card |

**What stays mocked through all 4 passes**: real Score backend, real on-chain action stream, real IRL weather API, wallet auth. All values are deterministic and inspectable; all data shapes are real and live in `lib/score/` (with `WeatherFeed` and `ActivityStream` to be added) so a real adapter swap is a binding-line change. [GROUNDED — `lib/score/index.ts:17`]
