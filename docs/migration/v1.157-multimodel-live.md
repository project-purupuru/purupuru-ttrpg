## What changed

### 1. The activation flag (cycle-107)

| Flag value | Bridgebuilder | Flatline | Red-team |
|-----------|---------------|----------|----------|
| `flatline_routing: true` (new default v1.157.0) | cheval | cheval | cheval |
| `flatline_routing: false` (rollback) | cheval (always — unconditional since cycle-103) | legacy `model-adapter.sh.legacy` | legacy |

**With the flag on**, all three consumers benefit from:

- **Chain-walk**: cheval walks the within-company `fallback_chain` (e.g. `gpt-5.5-pro → gpt-5.5 → gpt-5.3-codex → codex-headless`) on retryable errors.
- **Voice-drop**: when a voice's chain exhausts, flatline-orchestrator DROPS that voice from consensus instead of substituting another company's model.
- **MODELINV v1.1 envelope** at `.run/model-invoke.jsonl` recording `final_model_id`, `transport`, `config_observed`, `models_failed[]`, `models_requested[]` per invocation.

**With the flag off**, FL + RT fall back to the legacy `model-adapter.sh.legacy` path — no chain-walk, no voice-drop, no MODELINV v1.1. BB is unaffected.

To override per-operator in `.loa.config.yaml`:

```yaml
hounfour:
  flatline_routing: true   # default since v1.157.0
  feature_flags:
    flatline_routing: true
```

### 2. Within-company fallback chains (cycle-104)

Every primary in `model-config.yaml` now declares a `fallback_chain`. Example for Anthropic:

```yaml
providers:
  anthropic:
    models:
      claude-opus-4-7:
        fallback_chain:
          - anthropic:claude-opus-4-6
          - anthropic:claude-sonnet-4-6
          - anthropic:claude-headless
```

Cheval walks the chain on retryable errors (EmptyContent, RateLimited, ProviderOutage, RetriesExhausted, ContextTooLarge). The chain is within-company by design — substituting another company's model on partial failure was the cycle-102 T1B.4 anti-pattern, now structurally retired.

### 3. Headless mode (cycle-104)

Set `LOA_HEADLESS_MODE` to route the chain through CLI binaries instead of HTTP APIs:

| Mode | Chain effect | Use case |
|------|-------------|----------|
| `prefer-api` (default) | HTTP entries first, CLI fallback after | API keys present, normal operation |
| `prefer-cli` | CLI entries first, HTTP fallback after | Have both API keys + CLI subscriptions; route through subscription when possible |
| `api-only` | Strip CLI entries; HTTP only | Sandbox / CI without CLI binaries |
| `cli-only` | Strip HTTP entries; CLI only | Zero-API-key environment (e.g., fresh laptop, secure airgap) |

CLI binaries required for headless mode: `claude` (Claude Code), `codex` (OpenAI Codex CLI), `gemini` (Google Gemini CLI). See `grimoires/loa/runbooks/headless-mode.md`.

### 4. Voice-drop semantics (cycle-104 T2.8)

If a voice's entire within-company chain exhausts, the flatline orchestrator drops that voice from consensus and emits a `consensus.voice_dropped` event to the trajectory log. Consensus proceeds with the remaining voices. Previously (cycle-102 T1B.4), the orchestrator substituted another company's model in the dropped slot — that pattern is retired.

### 5. MODELINV v1.1 audit envelope (cycle-104 T2.6)

Every cheval invocation now emits a signed envelope at `.run/model-invoke.jsonl`. Fields:

```jsonc
{
  "primitive_id": "MODELINV",
  "event": "model.invoke.complete",
  "payload": {
    "models_requested": ["openai:gpt-5.5-pro", "openai:gpt-5.5", "openai:gpt-5.3-codex", "openai:codex-headless"],
    "models_failed": [/* per-entry failures if chain walked */],
    "final_model_id": "openai:gpt-5.5-pro",
    "transport": "http",                  // or "cli" for headless mode
    "config_observed": {
      "headless_mode": "prefer-api",
      "headless_mode_source": "default"   // env / config / default
    }
  },
  "signature": "...",                     // L1 audit-envelope signing
  "signing_key_id": "...",
  "prev_hash": "..."
}
```

This is the canonical evidence file. Diagnostic queries:

```bash
# Did this invocation walk the chain?
jq 'select(.payload.final_model_id != .payload.models_requested[0])' .run/model-invoke.jsonl

# All MODELINV envelopes from today
jq -c 'select(.primitive_id == "MODELINV")' .run/model-invoke.jsonl
```

### 6. Beads migration repair tool (cycle-105)

If you've been hitting `NOT NULL constraint failed: dirty_issues.marked_at` on `br ready` / `br create` / `br sync`, run:

```bash
tools/beads-migration-repair.sh

# Or via the wrapped health-check:
.claude/scripts/beads/beads-health.sh --repair
```

The tool heals dirty `.beads/beads.db` files in-place via SQLite's canonical recreate-and-swap pattern. Idempotent + reversible + backup-before-mutation.

### 7. Zone manifest (cycle-106, closes #818)

New `grimoires/loa/zones.yaml` declares framework-zone vs project-zone vs shared paths. The framework's own operator history (cycles/, NOTES.md, handoffs/, etc.) is now project-zone and gitignored — no longer leaks into downstream installs via `/update-loa`.

New operators get an empty project-zone scaffolding via `mount-loa`. Existing operators keep their working tree intact; nothing destructive happens.

See `grimoires/loa/runbooks/zone-hygiene.md` for full operator runbook.

## Recommended verification post-upgrade

1. **Check the flag**:
   ```bash
   yq -r '.hounfour.flatline_routing // false' .loa.config.yaml
   # Expected: true
   ```

2. **Smoke a multi-model run**:
   ```bash
   echo "test prd" > .run/smoke.md
   .claude/scripts/flatline-orchestrator.sh --doc .run/smoke.md --phase prd --dry-run
   # Expected: clean dry-run without errors
   ```

3. **Inspect MODELINV envelope**:
   ```bash
   tail -1 .run/model-invoke.jsonl | jq '.payload.transport, .payload.final_model_id'
   ```

4. **Check beads health**:
   ```bash
   .claude/scripts/beads/beads-health.sh --json | jq '.status'
   # Expected: HEALTHY (or DEGRADED — both fine; not MIGRATION_NEEDED)
   ```

## Rollback

If the new defaults cause problems:

```yaml
# .loa.config.yaml
hounfour:
  flatline_routing: false   # revert FL + RT to legacy model-adapter.sh.legacy
```

Then `bash -c 'source .claude/scripts/model-adapter.sh; is_flatline_routing_enabled || echo DISABLED'` should print DISABLED.

BB cannot be rolled back to pre-cheval state without a larger revert; the Node-fetch adapter registry was retired in cycle-103 and would need restoration.

## Known limitations

- **Voice-drop production proof**: the mechanism is bats-tested with stubbed cheval; we don't have a production trajectory entry showing it fired on a real chain-exhaustion event (KF-003 didn't reproduce in cycle-104 T2.10 live replay). If KF-003 returns, voice-drop will engage; until then, the bats integration tests are the canonical evidence.
- **Strace assertions skip on hardened kernels**: `tests/e2e/test_cli_only_zero_api_key.bats` skips T2.11-1/T2.11-2 when `kernel.yama.ptrace_scope >= 2`. T2.11-3 audit-envelope cross-check provides the load-bearing structural proof either way.
- **Pre-existing test failures**: 3 `test_flatline_routing.py` failures + 1 `flatline-orchestrator-max-tokens.bats:140` expectation drift + 1 `__tests__/persona.test.ts` rate-limit-retry mock failure. All pre-existing on main; not introduced by this release.
