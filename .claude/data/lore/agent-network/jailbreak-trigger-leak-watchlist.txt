# Trigger-leak watchlist for cycle-100 NFR-Sec1 lint.
#
# Lines starting with `#` are comments. One regex per non-comment line.
# Patterns are case-insensitive (POSIX BRE/ERE; the lint script uses
# `grep -iEH` so escaping follows ERE rules).
#
# Inclusion criterion: a string is on the watchlist iff it is high-confidence
# adversarial (would be picked up by a security-engineer code review) AND
# narrowly enough scoped that we won't false-positive on commodity code or
# existing in-tree documentation that legitimately discusses prompt injection
# as an abstract topic.
#
# When adding entries: include a brief comment block above the regex citing
# the source (OWASP-LLM-01 / DAN / Anthropic / cycle-098 sprint-N finding).
# Vectors must construct these strings at runtime in fixtures; this lint blocks
# verbatim source-file leaks of the same strings outside the corpus tree.
